# Final Demo Video Script

Purpose: live recording script for the final demo. This is not a PPT-style video plan. The recording should show the runtime system: natural-language command, L1/Qwen bridge, structured IntentPacket, target marker in Gazebo/RViz, and robot arm motion through the RL `Approach -> Finisher` path.

## Video Goal

Show the modular robot learning pipeline running live:

```text
Natural language command
-> Qwen / L1 bridge
-> structured IntentPacket
-> RL skill request
-> target marker in Gazebo / RViz
-> Approach -> Finisher runtime
-> ROS2 / Gazebo arm motion
```

Important scope statement for the narration:

```text
This is a Phase 3A sim-validation demo. It does not claim full holder1-to-holder8 kitchen transport is solved. It demonstrates the L1-to-RL-to-Gazebo runtime path, target visualization, and learned local motion.
```

## Scene Setup

Recommended recording layout:

- Left side: terminal with large font.
- Right side: Gazebo GUI or RViz.
- Optional: keep VS Code closed or minimized so the recording focuses on the system.

Recommended live commands:

```bash
bash scripts/final/run_live_gz_screen_recording_demo.sh local_skill
bash scripts/final/run_live_gz_screen_recording_demo.sh tray_like_transport
```

For the cleanest recording, start Gazebo first in a separate terminal, then run
the live demo command. The wrapper now defaults to `LAUNCH_SCENE=0`, so it does
not restart or close Gazebo during the recording.

The command uses the larger `workspace_showcase` profile by default:

```text
--max-targets 8
--target-profile workspace_showcase
--approach-steps 72
--finisher-steps 10
```

This profile is intentionally presentation-oriented. It should make the arm
motion obvious in Gazebo. A target may fail; the goal is to show the live
L1-to-RL-to-Gazebo path and a larger movement range, not to claim every
workspace stress target is solved.

The final video should contain two demos:

- Demo A: `local_skill`, showing the expanded local RL workspace.
- Demo B: `tray_like_transport`, showing a mock tray-carry semantic waypoint plan executed by the same RL stack.

If GUI is not available, use the headless validation path and show terminal logs / output artifacts. Do not rely on the fixed camera-topic video as the main demo because the scene cameras may have poor viewpoints.

```bash
bash scripts/final/run_live_gz_vlm_demo.sh local_skill
```
